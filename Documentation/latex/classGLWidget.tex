This function sets up the G\+L object. 

In initialisation the vertex buffers will be constructed. Each prism of the system is actually made up of 6 nodes, the ids rangeing from 0-\/5. For correct lighting, same nodes should hold different normals at the edges. Therefore, for each prism of 6 nodes, a total of 18 vertices are stored\+: 3 upper surface, 3 lower surface, 4 for each of 3 side rectangles. The indice order is stored in the array vertex\+Order\+To\+Add\+For\+Prism.

For each of the stored vertices, first the position array is inserted into the G\+L\+Widget\+::\+G\+L\+Vertices.

then the current colour of the vertex,

then the current normal pointing out of the vertex,

and lastly, the unique identifier colour of Element\+Colour are appended to the vertex buffer.

Each prism will be defined by size trienagles (upper/lower surfaces and two triangles each for each of the three side rectangles). These triangles will be defined by the 18 vertices added to the vectex buffer for this triangle. Their order in terms of the elemental vertex index is stored in triangles\+For\+Node\+Order\+Added\+By\+Element.

As these indices are in the elemental definition, they are be positioned in the vertex array with the offset\+: (element index from the simulation) $\ast$18. With the offset correction, the tirangle information is appended to the G\+L\+Widget\+::\+G\+L\+Elements

Then the vertex (G\+L\+Widget\+::vbo) and element (G\+L\+Widget\+::veo) buffers are generated with the size of the vectors G\+L\+Widget\+::\+G\+L\+Vertices and G\+L\+Widget\+::\+G\+L\+Elements.

The shaders are set up.

In development, the shader status can be checked in a hard coded loop within this function. The loop is not active in release, but can be activated with setting check\+Shader\+Status = true insisde the code. This loop will also print the log buffer.

The position attribute is assiged for the first 3 items on the stride of 12 floats in the vertex buffer.

The colour attribute is assiged after an offset of 3 floats (for position) to the next 3 items on the stride of 12 floats in the vertex buffer.

The normal attribute is assiged after an offset of 6 floats (for position + colour) to the next 3 items on the stride of 12 floats in the vertex buffer.

Fianllt, the unique colour identifier of the elementm picking colour attribute is assiged after an offset of 9 floats (for position + colour + normal) to the next 3 items on the stride of 12 floats in the vertex buffer.

This function checks for selection of an element under the last mouse click, updates model as needed. 

This function starts by drawing the schene into the buffer with the \hyperlink{classGLWidget_a218fadfeda519fd8a120b702f04771c8}{G\+L\+Widget\+::draw\+For\+Picking} function. Here, all visible elements of the schene are drawn as usual, but instead of colourd defiend by teh user inputs, they are drawn with their unique drawing colours. Please bear in mind this is a transition phase for screen picking, and thse colours are not displatyed at any point.

The picking attempt is registered bu the function \hyperlink{classGLWidget_ab992c4c25439a5ef23031991015451c1}{G\+L\+Widget\+::mouse\+Release\+Event}, and the last pointwhere the mouse was release is recorded in Last\+Pos, an input of this function. Picked colour is read into G\+L\+Widget\+::\+Picked\+Colour via function \hyperlink{classGLWidget_afc09ab8fa7a8163ca1b69e7953e33dc4}{G\+L\+Widget\+::get\+Colour\+Of\+Point}.
